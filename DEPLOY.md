# AB 看板部署说明

这份工程已经准备成两种分享方式：

1. 局域网分享
2. 公网静态部署

## 方式一：局域网分享

适合先让同事试用。

1. 双击 `start-share-dashboard.cmd`
2. 页面会在你电脑本机打开
3. 终端会打印局域网地址，例如 `http://192.168.x.x:8123/`
4. 把这个地址发给同一网络里的同事

注意：

- 你的电脑必须保持开机
- 第一次运行如果 Windows 防火墙弹窗，请允许 Node.js 通过“专用网络”
- 如果同事无法打开，先确认你们在同一个网络里

## 方式二：公网静态部署

这个页面现在已经可以直接放到任何静态托管平台。

要上传的最小文件：

- `index.html`
- `ab-dashboard.html`
- `ab-dashboard.css`
- `ab-dashboard.js`

说明：

- `index.html` 会自动跳转到 `ab-dashboard.html`
- `ab-dashboard.html` 会优先读取同级目录下的本地依赖文件
- 如果本地依赖不存在，会自动回退到 CDN 加载 `xlsx` 和 `html2canvas`

可用的平台：

- Vercel
- Netlify
- GitHub Pages
- 腾讯云 COS 静态网站
- 阿里云 OSS 静态网站
- 任意 Nginx 静态目录

## 我还需要你提供什么

如果你要我继续把它真正发出去，我只需要你告诉我下面三件事中的一件：

1. 你想先走“局域网分享”，那你只需要运行 `start-share-dashboard.cmd`
2. 你想走“公网部署”，请告诉我你要用哪个平台
3. 如果你要绑定正式域名，请把域名和你能登录的平台告诉我

## 推荐顺序

1. 先用局域网分享给 1-2 个同事试用
2. 确认没有识别和图表问题后
3. 再部署到公网并绑定域名

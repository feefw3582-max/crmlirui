# MPL: AB 实验可视化产品 Prompt 清单

> MPL = Message Prompt List
>
> 来源：当前 Codex 对话线程
>
> 更新时间：2026-04-01
>
> 用途：沉淀用户真实需求，后续继续迭代时可直接作为统一输入

## 1. 合并后的主 Prompt

请持续把这个产品做成一个真实可用的 AB 实验数据可视化页面，而不是只做演示稿。页面需要支持用户上传真实 Excel/CSV 底表，自动识别实验字段、日期、组别、维度字段和统计指标，不强依赖固定列名；支持用户手动纠偏、选择对照组和实验组、配置统计量公式、做趋势图和属性拆解，并能导出截图。交互和图表表现要向飞书/钉钉这类智能办公产品靠齐，优先解决真实使用时的 bug、字段识别、图表体验、维度筛选和部署分享问题。后续新增需求时，继续追加到本 MPL 中并优先按最新用户要求更新产品。

## 2. 稳定约束

- 目标不是代码演示，而是做出一个用户能直接使用的简陋但可工作的前端页面。
- 不能强制要求上传文件必须使用固定字段名，系统需要尽量自行识别。
- 真实业务里会有多实验组、多对照组、多维度、多统计量和派生公式。
- 趋势图、属性拆解、智能识别是高频改动区，后续迭代优先保证它们的可用性。
- 用户给出的新约束默认覆盖旧约束中的冲突部分。

### MPL-004

### MPL-005
- 原始 prompt：你的"智能识别“和”趋势图标“两个板块bug过多，功能和交互也非常的简陋，你现在调研一些智能办公软件的图表和多维表格（如飞书，钉钉），然后修整bug，迭代功能和交互
- 整理后意图：以飞书、钉钉风格为参考，重点修智能识别和趋势图的 bug、交互和观感。
- 模块：竞品参考、趋势图、智能识别
- 优先级：P0

### MPL-006
- 原始 prompt：你要让用户可以选择，如果有年级的话，我是不是可以只要初中高中的，大学的就不要了。如果有学科，我是不是只要文科，理科我就可以不要。如果有五个组，我是不是可以选择哪个是实验组（比如输入110，108）哪个是对照组（比如输入33782，33783，输入两组的话，对照组的数据就是这两组的平均值，而像转化率这类数据的话就是总的订单量除以总的访问人数（uv/pv，你需要在智能识别模块中对于这些需要计算的属性，在后面有一个默认计算公式，而用户可以对其进行修改））你的可视化趋势图标必须向飞书靠齐，而且我试着上传了我desktop上的文件“tid维度cvr”，你的这个页面居然无法处理它
- 整理后意图：支持维度值筛选、实验组/对照组手动指定、多对照组聚合、派生指标重算、公式可编辑，并且必须处理真实上传文件。
- 模块：维度筛选、组选择、公式系统、真实文件兼容
- 优先级：P0

### MPL-007
- 原始 prompt：属性拆解里新增可以自行选择组别和添加属性。趋势图标新增可以自己加自变量和因变量
- 整理后意图：属性拆解支持自选组别与多属性组合；趋势图区支持自定义 X/Y 轴字段。
- 模块：属性拆解、趋势图配置
- 优先级：P1

### MPL-008
- 原始 prompt：“默认计算公式”模块你应该改一下，不是只有转化率和arpu，把它放上去一点，放到维度值筛选上面，让用户自行选择或者手动添加没有准备的统计量，比如点击率，转化率，GMV，ARPU，置信度（缺少变量就弹出小行红色提醒）等等，选择过后我们可以让用户自行为这个统计量配置公式，这个公式可以是普通的四则运算，（+,-,*,/),也可以是VLOOKUP函数。维度选择那里也要改，我们除了学科和年级，用户可能以后还会有别的维度，所以我们应该让用户自行选择表中我们识别出的维度（也就是表头），然后再有这样子在维度里面选需要的类别。检查你的“已识别shema”模块，这个地方有ui组件的重叠遮挡情况，改成左右滑动
- 整理后意图：公式工作台要支持更多预置统计量和自定义统计量，支持四则运算和 VLOOKUP；维度筛选要支持任意识别出的维度字段；schema 展示区改为横向滚动避免遮挡。
- 模块：公式系统、维度选择、schema UI
- 优先级：P1

### MPL-009
- 原始 prompt：趋势图标部分新增可选折线/饼图/柱状图/，新增当系统无法配置y轴刻度时（全是1或者0），取这五个刻度：0，该属性中出现的最大值和最小值，以及它们之间的四分之一，四分之二，四分之三数）
- 整理后意图：趋势图区支持折线图、柱状图、饼图；在常量或极低跨度场景下启用兜底 Y 轴刻度。
- 模块：趋势图图形类型、Y 轴刻度
- 优先级：P1

- 模块：部署、分享
- 优先级：P0

### MPL-012
- 原始 prompt：有两个点我们需要改一下：首先，用户上传的表格里，有很多“all”属性的数据是也放在里面的，这就导致我们会把这一块给多算进去，你记得把subject和grade两个属性中的“all”给踢出去。其次，我们需要在趋势图表这个板块做一个功能，在左上角加一个滑动变比值器，用户滑动这个东西，折线图的y轴的刻度比值会发生相应的变化。×2的时候，我们的0刻度值会变成最大刻度值的二分之一，×3的时候，会变成三分之二。滑动变比值器的选择是0.1倍到10倍，中点处是1倍，中点左边是0-1的倍数，右边是1-10，两头一样长，方便用户更灵敏地去选倍数，这个前端交互动画一定是流畅的
- 整理后意图：过滤 `subject` 和 `grade` 中的 `all` 聚合行；趋势图增加 0.1x-10x 的 Y 轴倍数滑块，中点 1x，拖动体验要平滑。
- 模块：数据过滤、趋势图增强
- 优先级：P0

## 4. 后续维护规则

- 新 prompt 继续按 `MPL-XXX` 递增追加。
- 每条新增需求都保留“原始 prompt”和“整理后意图”两层。
- 如果新需求与旧需求冲突，以最新编号为准，并在该条下显式说明覆盖范围。
- 继续开发时，优先参考“合并后的主 Prompt”和 P0 条目。
